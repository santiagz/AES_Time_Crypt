How encryption works

step by step
  1. Your file bytes arrive at the server over HTTPS
  2. Server generates a fresh AES-256-GCM key + random 96-bit nonce
  3. File is encrypted â†’ ciphertext. Plaintext is immediately discarded from memory
  4. Key is stored in the vault with your unlock timestamp. Only the key is stored â€” not your file
  5. Response: a .tlp file containing key_id + ciphertext. This is what you keep
  6. Server returns the key only after the unlock date â€” no exceptions, no admin override

The .tlp file holds only ciphertext + a key reference ID. Even if someone intercepts your .tlp, they cannot decrypt it without the server releasing the key.
